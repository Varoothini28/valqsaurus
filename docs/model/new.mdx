## 2. Create a Simple Model


To build a Simple Model first, you need to import data into Power BI. This step is followed by creating a
simple model on ValQ.

The steps have been already covered under <Link to={useBaseUrl('docs/intro/model')}>Using ValQ - An Example</Link> section. 
Or you can refer <a href="https://valq.com/wp-content/uploads/valq-for-microsoft-power-bi-beginners-tutorial.pdf" target="_blank">this</a> pdf for step by step instructions on how to get started to build a Simple Model. <br />


A Simple model looks like this.

<div style={{textAlign: 'center'}}>
  <Zoom>
<img alt="New Tab" src={useBaseUrl('/doc-images/model/new10.png')} />
  </Zoom>
</ div>

#### Edit a Simple Model

<div style={{textAlign: 'justify'}}>
When a node is selected, you will be able to only view the model with Parent
and Children nodes and you cannot perform any
configuration on it. But you can generate a copy of the Parent node
and proceed with Configuration based on your choice (see Figure
below).
</ div> <br />

<div style={{textAlign: 'center'}}>

#### Dynamic Scaling

<div style={{textAlign: 'justify'}}>
Based on your data source, the model will get created and
the Value Display (scaling) for all the nodes will be updated appropriately based
on the data source (see Figure below). For our example, the Number
Scaling is “0m”.
</ div> <br />

The time aggregation of the Parent node is always set to Sum of Periods of Child nodes on a Simple Model.
